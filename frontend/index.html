EDGE SECURITY ARCHITECTURE

Secure AWS Static Website Architecture

I hardened the static site path around CloudFront, private S3 origins, and AWS WAF instead of leaving the bucket as the main public surface.

The main design choice was to keep direct origin access closed and let managed WAF rules handle common edge threats before traffic reached the site.

What I would improve next: connect WAF logs to monitoring and add policy checks before Terraform changes are applied.

SERVERLESS COMMUNICATION FLOW

Serverless Contact Form API

I built the contact form as a small serverless flow with API Gateway, Lambda, SES, and CloudFront.

I focused on the boring but important parts: validation before Lambda, useful logs for failed requests, and edge protections to slow down spam.

What I would improve next: add queue-based retries so email delivery failures are not lost after a single Lambda attempt.

IDENTITY-AWARE API PLATFORM

Secure Serverless Backend API

I put together a serverless API using Cognito, JWT authorization, API Gateway, Lambda, and Terraform.

The goal was to keep identity checks at the API boundary, limit what each Lambda function can access, and log enough context to debug failures.

What I would improve next: add tracing across API Gateway and Lambda, then test how the service behaves under regional failure scenarios.

ENGINEERING APPROACH

How I Approach Cloud Engineering

I try to build cloud systems that are understandable after the first deployment, not just impressive in a diagram. That usually means small permissions, repeatable infrastructure, clear logs, and enough monitoring to explain what happened when something breaks.
